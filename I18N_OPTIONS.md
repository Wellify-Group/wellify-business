# Варианты автоматизации переводов

## Текущая ситуация
У вас уже есть рабочая система переводов через `useLanguage()` и `TRANSLATIONS`. Она работает хорошо, но требует ручного добавления переводов.

## Варианты решения

### 1. ✅ Рекомендуется: Остаться на текущей системе + добавить автогенерацию

**Плюсы:**
- Не нужно переписывать код
- Полный контроль над переводами
- Можно добавить автоматическую генерацию недостающих переводов

**Что нужно:**
1. Установить пакет для перевода:
   ```bash
   npm install @vitalets/google-translate-api
   # или
   npm install deepl-node
   ```

2. Обновить `scripts/auto-translate.ts` для использования реального API

3. Добавить недостающие ключи в русскую версию, запустить скрипт

**Использование:**
```bash
npx tsx scripts/auto-translate.ts
```

---

### 2. Миграция на `next-intl` (более стандартное решение)

**Плюсы:**
- Стандартное решение для Next.js
- Лучшая поддержка TypeScript
- Встроенная поддержка роутинга по языкам
- Больше инструментов и плагинов

**Минусы:**
- Нужно переписать весь код переводов
- Нужно изменить структуру файлов
- Больше зависимостей

**Установка:**
```bash
npm install next-intl
```

**Пример использования:**
```tsx
import {useTranslations} from 'next-intl';

export default function Component() {
  const t = useTranslations('Login');
  return <h1>{t('title')}</h1>;
}
```

---

### 3. Использовать сервисы перевода (Crowdin, Lokalise, Phrase)

**Плюсы:**
- Профессиональные переводы
- Работа с переводчиками
- Автоматическая синхронизация
- Версионирование переводов

**Минусы:**
- Платные (есть бесплатные планы)
- Нужна интеграция
- Больше настройки

---

## Рекомендация

**Остаться на текущей системе** и добавить скрипт автогенерации переводов через API.

**Почему:**
1. Ваша система уже работает хорошо
2. Не нужно переписывать код
3. Можно добавить автоматизацию без больших изменений
4. Полный контроль над переводами

**Что сделать:**
1. Установить пакет для перевода (Google Translate API или DeepL)
2. Обновить `scripts/auto-translate.ts`
3. При добавлении новых ключей запускать скрипт для автогенерации

---

## Быстрый старт с автогенерацией

1. Установите пакет:
   ```bash
   npm install @vitalets/google-translate-api
   ```

2. Обновите `scripts/auto-translate.ts`:
   ```typescript
   import translate from '@vitalets/google-translate-api';
   
   async function translateText(text: string, targetLang: 'en' | 'uk'): Promise<string> {
     const res = await translate(text, { to: targetLang === 'uk' ? 'uk' : 'en' });
     return res.text;
   }
   ```

3. Запустите:
   ```bash
   npx tsx scripts/auto-translate.ts
   ```

**Примечание:** Google Translate API бесплатный, но с ограничениями. Для продакшена лучше использовать DeepL API (платный, но качественнее).
