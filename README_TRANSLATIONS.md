# Автоматическая генерация переводов

Система автоматически генерирует недостающие переводы для английского и украинского языков на основе русской версии.

## Как это работает

1. **Источник переводов**: Русский язык (`ru`) является основным источником
2. **Целевые языки**: Английский (`en`) и Украинский (`ua`)
3. **Автоматизация**: При изменении `lib/translations.ts` автоматически генерируются недостающие переводы

## Использование

### Ручной запуск

```bash
npm run translate
```

### Автоматический запуск

Система автоматически запускается при коммите изменений в `lib/translations.ts` через git hook.

## Процесс работы

1. Вы добавляете новый ключ перевода в русскую версию (`ru`)
2. При коммите git hook проверяет изменения
3. Скрипт находит недостающие переводы для `en` и `ua`
4. Автоматически генерирует переводы через Google Translate API
5. Сохраняет обновленный файл `lib/translations.ts`

## Пример

**До:**
```typescript
ru: {
  new_key: "Новый ключ",
},
en: {
  // new_key отсутствует
},
ua: {
  // new_key отсутствует
}
```

**После запуска `npm run translate`:**
```typescript
ru: {
  new_key: "Новый ключ",
},
en: {
  new_key: "New key", // Автоматически переведено
},
ua: {
  new_key: "Новий ключ", // Автоматически переведено
}
```

## Технические детали

- **API перевода**: Google Translate (через `@vitalets/google-translate-api`)
- **Задержка между запросами**: 100ms (чтобы не перегружать API)
- **Обработка ошибок**: При ошибке перевода возвращается оригинальный текст

## Ограничения

- Google Translate API имеет лимиты на количество запросов
- Качество перевода может требовать ручной проверки для важных текстов
- Рекомендуется проверять автоматически сгенерированные переводы перед продакшеном

## Отключение автоматизации

Если нужно отключить автоматическую генерацию, удалите или переименуйте файл `.husky/pre-commit`.
